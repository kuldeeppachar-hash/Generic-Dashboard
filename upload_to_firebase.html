⬆ Sales Dashboard — Firebase Data Uploader

Upload the raw SAP export → it splits rows into Generic and C.Care divisions, aggregates each separately, and pushes both live to Firebase in one go. No SO data is generated or stored.
// SOURCE OF TRUTH
This tool reads only the raw SAP-derived export Excel — nothing else feeds this dashboard.
It auto-detects the data sheet (some exports have blank scratch sheets before the real one) and the column format.
SAP direct export (has a Division Name column): rows where Division Name = "Generic" go to the Generic tile. Rows where Division Name starts with "C.Care" (e.g. "C.Care (Anaesthesia)", "C.Care (Antibiotics)") go to the C.Care tile, further split by the text in brackets as subtype.
Distributor-type export (older format — Distribtor Type / Product type columns only): this schema has no real division field, so it can only ever produce Generic data. C.Care is not recoverable from this format. "City" is read from City or Head Quarter, whichever the file has.
// MONTH-BY-MONTH STORAGE
Each upload is detected by its Invoice Date and saved under its own month — it no longer overwrites other months.
Generic and C.Care are stored as sibling nodes per month (/months/{month}/generic/... and /months/{month}/ccare/...) — uploading one doesn't touch the other.
Re-uploading a file for a month/division you've already uploaded will overwrite that month+division only (useful for corrections); everything else stays untouched.
If a file has zero C.Care rows for a month, the C.Care node for that month is left as-is (not wiped) — so a Generic-only re-upload never deletes existing C.Care data.
